# 中国平安专属做T策略 v1 补充设计

## 1. 背景

本文档补充 `t-alpha` 当前做T策略的产品和回测口径，用于后续实现“中国平安专属做T策略 v1”。现有代码已有策略、回测、邮件和默认监控标的的基础骨架，但尚未完成真实 3 年回测、参数微调、最近 3 个月样本外展示，以及传入股票代码后生成专属策略和监控任务的完整闭环。

本策略仅用于量化研究和交易提醒，不自动下单，不构成投资建议。

## 2. 策略目标

策略目标是对单只 A 股生成低吸型做T方案。v1 首个测试标的是 `601318.SH` 中国平安，后续通过外放接口传入任意 A 股代码，触发该股的策略生成、回测、报告和监控逻辑。

核心目标：

- 只支持“先买后卖底仓”的低吸型做T。
- 默认用户已有底仓，且底仓金额大于单次做T金额；v1 不校验底仓。
- 单次做T金额在 `5000` 到 `20000` 元之间，根据买点质量动态调整。
- 每只股票每日最多开一笔做T交易。
- 单笔收益率达到 `> 3%` 后卖出，计为做T成功。
- 最长持仓 `10` 个交易日；若第 10 个交易日仍未达到 `> 3%` 收益，则卖出并计为做T失败。
- v1 暂不设置中途止损。
- 最近 3 年数据用于回测和参数微调。
- 最近 3 个月作为样本外结果展示，输出符合条件的买点和实现收益。

## 3. 交易定义

### 3.1 开仓方向

v1 只做低吸型做T：

1. 策略在盘中或回测中识别低吸买点。
2. 先买入一笔 T 仓。
3. 后续当收益达标时，卖出已有底仓中等量股票，实现对当次 T 仓的回转交易归因。

### 3.2 仓位约束

v1 假设用户底仓充足：

- 底仓金额默认大于做T金额。
- 不检查账户持仓。
- 不检查可用资金。
- 不处理底仓不足导致无法卖出的场景。

后续版本可增加账户持仓输入和可交易股数校验。

### 3.3 单次金额

单次金额范围：

- 最小：`5000` 元
- 最大：`20000` 元

金额由信号评分决定。评分应至少考虑：

- 回测成功率
- 扣成本后平均单笔收益
- 最近 3 个月表现
- 最大连续失败次数
- 当前买点与历史优质买点的相似度

买入股数按 A 股 `100` 股整数手向下取整。不足 `100` 股则不生成买入提醒。

### 3.4 持仓重叠

v1 不允许同一只股票存在重叠 T 仓：

- 一笔 T 仓开仓后，在该笔交易卖出前，不再开新的 T 仓。
- 回测和实盘监控都按该规则执行。
- 这样可以保持收益归因、成功率和邮件提醒逻辑清晰。

## 4. 成交和退出口径

### 4.1 买入成交价

为避免未来函数，回测中信号和成交必须分离：

- 信号在某根 60 分钟 K 线收盘后确认。
- 买入价使用下一根 60 分钟 K 线开盘价。

实盘提醒中，邮件可展示当前参考价和建议买入区间，但回测统计不得使用信号 K 线后续尚不可知的最低价或收盘后信息。

### 4.2 卖出成交价

卖出条件只有两个：

1. 持仓期间价格达到或超过目标价 `买入价 * (1 + target_return)`，其中 `target_return` 默认 `0.03`，视为收益达标，按目标价卖出，计为成功。
2. 若持仓满 `10` 个交易日仍未达到收益目标，则按第 10 个交易日收盘价卖出，计为失败。

v1 不设置盘中止损，不提前按亏损退出。

### 4.3 交易成本

v1 默认交易成本按万 1 计算，并预留配置项。

默认实现口径：

- `trade_cost_rate = 0.0001`
- 回测统计中按一笔完整做T交易扣除该成本。

配置预留：

- 后续可扩展为买入佣金、卖出佣金、印花税、滑点、最低佣金等分项。
- 若后续接入更真实的 A 股成本模型，应在报告中明确展示成本假设。

## 5. 回测设计

### 5.1 数据周期

回测使用最近 3 年数据：

- 日线：用于趋势、波动、流动性等过滤。
- 60 分钟线：用于买点识别、开仓和持仓期间收益判断。

数据需要使用前复权价格，以减少分红送转对回测的影响。

### 5.2 数据划分

3 年数据分为：

- 最近 `3` 个月：先从 3 年窗口末端保留出来，作为最终样本外展示区间。
- 剩余历史数据前 `70%`：训练和参数微调。
- 剩余历史数据后 `30%`：验证。

参数优化不得使用最近 3 个月展示区间的结果倒推规则。

### 5.3 参数微调目标

参数微调排序优先级：

1. 成功率
2. 扣成本后平均单笔收益
3. 验证集成功率
4. 最大连续失败次数

成功率是硬门槛，但不能作为唯一指标。若成功率高但平均单笔收益为负，或最大连续失败次数过高，策略不能进入监控。

最近 3 个月结果只在参数冻结后用于最终展示和健康检查，不参与参数搜索。

### 5.4 最低样本数标准

结合公开量化回测常见口径和单股做T的现实频率，v1 使用分级样本标准：

- 3 年有效交易数 `>= 60` 笔：策略可用，可进入监控候选。
- 3 年有效交易数 `30` 到 `59` 笔：仅作为观察策略，不自动进入监控。
- 3 年有效交易数 `< 30` 笔：策略无效，不生成监控任务。

最近 3 个月不设置硬性交易笔数门槛，只展示实际出现的买点。若最近 3 个月有效交易数 `< 3` 笔，报告中标记为“近期样本不足，只能观察”。

取 `60` 笔作为可用门槛的原因：

- `30` 笔通常只能支持非常初步的统计判断。
- `100` 笔以上更适合高频或多标的策略，但对单只股票、每日最多一次、最长持仓 10 个交易日的做T策略可能过严。
- `60` 笔约等于 3 年每年 20 笔、每月 1 到 2 笔，既避免过度交易，也能降低少量样本造成的虚高胜率。

## 6. 成功标准

策略进入监控候选必须同时满足：

- 3 年有效交易数 `>= 60`。
- 3 年做T成功率 `> 50%`。
- 验证集成功率 `> 50%`。
- 扣成本后平均单笔收益 `> 0`。
- 最近 3 个月未明显失效；若样本不足，需要在报告中明确提示。

策略排序时，成功率越高越优；若成功率接近，则比较扣成本后平均单笔收益、最近 3 个月表现和最大连续失败次数。

## 7. 最近三个月报告

最近 3 个月报告需要输出每一笔符合条件的交易明细：

- 股票代码
- 信号时间
- 买入时间
- 买入价
- 买入金额
- 买入股数
- 卖出时间
- 卖出价
- 持仓交易日数
- 收益率
- 扣成本后收益率
- 收益金额
- 成功或失败
- 触发原因

报告还需要输出汇总指标：

- 交易笔数
- 成功笔数
- 失败笔数
- 成功率
- 平均单笔收益
- 中位数单笔收益
- 最大单笔亏损
- 最大连续失败次数
- 近期样本是否充足

## 8. 外放接口设计

v1 需要预留两个接口方向。

### 8.1 生成策略和报告

接口用途：传入 A 股代码，生成该股专属做T策略、完成 3 年回测和参数微调，并返回最近 3 个月买点明细和汇总表现。

建议接口：

```text
POST /api/v1/strategy/t0/build
```

请求字段：

```json
{
  "code": "601318.SH"
}
```

响应至少包含：

- 股票代码
- 策略名称
- 参数配置
- 3 年回测摘要
- 验证集摘要
- 最近 3 个月交易明细
- 是否达到监控候选标准
- 未通过原因
- 风险提示

### 8.2 加入监控

接口用途：将已生成且通过门槛的策略加入监控列表，盘中检测买点并触发邮件提醒。

建议接口：

```text
POST /api/v1/strategy/t0/monitor
```

请求字段：

```json
{
  "code": "601318.SH",
  "enabled": true
}
```

未通过策略门槛的股票不得自动进入监控，除非后续明确增加“观察模式”。

## 9. 监控和邮件提醒

v1 监控逻辑：

- 默认以 `601318.SH` 中国平安作为测试标的。
- 每个交易日在固定检查点运行。
- 每日最多生成一次买入提醒。
- 若该股票已有未完成 T 仓，不再生成新的买入提醒。
- 信号达到策略条件时发送邮件。

邮件内容至少包含：

- 股票代码
- 信号时间
- 当前价格
- 建议买入区间
- 建议买入金额
- 建议买入股数
- 目标卖出价
- 最长持仓期限
- 历史成功率
- 3 年样本数
- 最近 3 个月表现摘要
- 触发原因
- 风险提示

## 10. 不在 v1 范围内

v1 暂不实现：

- 先卖后买回的高抛型做T。
- 中途止损。
- 多笔 T 仓重叠。
- 账户资金和底仓校验。
- 自动下单。
- 多股票组合资金分配。
- 盘中逐笔或 tick 级撮合。
- 印花税、滑点、最低佣金的真实分项成本模型。

## 11. 参考资料

公开资料中常见口径认为，少量交易样本容易导致胜率失真；约 30 笔只能做初步统计，100 笔以上更可靠，200 到 500 笔更适合严肃验证。考虑到本策略是单只 A 股、每日最多一次、最长持仓 10 个交易日，v1 采用 `60` 笔作为进入监控候选的现实下限。
